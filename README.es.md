# David Garcia - Portafolio Personal

Este es el repositorio del portafolio web personal de **David Garcia**, un Desarrollador Backend. El proyecto está construido utilizando [Astro](https://astro.build/), un framework web moderno y ultrarrápido diseñado para generar sitios web estáticos y dinámicos de alto rendimiento.

## 🚀 Tecnologías y Stack

En este portafolio destaco mi experiencia y conocimientos en el siguiente stack tecnológico:

- **Desarrollo Backend:** Java, Spring Boot, Node.js, Express, Python.
- **Bases de Datos:** PostgreSQL, SQL, MongoDB.
- **Herramientas & Sistemas:** Odoo, Docker, Git.
- **Frontend (Conocimientos Básicos):** HTML, JavaScript, React.

## 🧞 Comandos de Desarrollo

El proyecto está configurado para utilizar `pnpm` como gestor de dependencias. Todos los comandos deben ejecutarse desde la raíz del proyecto en tu terminal:

| Comando | Acción |
| :--- | :--- |
| `pnpm install` | Instala todas las dependencias necesarias. |
| `pnpm dev` | Inicia el servidor de desarrollo local en `localhost:4321`. |
| `pnpm build` | Compila el sitio para producción, generando los archivos en el directorio `./dist/`. |
| `pnpm preview` | Previsualiza la compilación de producción de forma local antes de desplegar. |

## 📁 Estructura del Proyecto

Dentro del proyecto de Astro, encontrarás la siguiente estructura principal de archivos:

```text
/
├── public/          # Archivos estáticos públicos (favicon, etc.) que no requieren procesamiento.
├── src/
│   ├── assets/      # Recursos gráficos y SVGs utilizados en el sitio.
│   ├── components/  # Componentes de UI reutilizables (Profile.astro, Stack.astro, Projects.astro, etc.).
│   ├── layouts/     # Componentes de diseño estructural que envuelven las páginas.
│   └── pages/       # Archivos de enrutamiento. Aquí se encuentra el `index.astro` (página principal).
└── package.json     # Configuración de scripts y dependencias del proyecto.
```
